Part 1 · Demo 端 AI 能力（本地实现，无需 API Key）
Demo

6 项 AI 能力，全部在浏览器本地运行

Demo 不依赖网络或大模型 API，所有 AI 效果由本地规则引擎（anchornote-embed.js）实现，确保评委在任何网络环境下都能完整体验产品闭环。

🏷️ 内容理解与摘要生成 Demo + 插件
用户选中文字保存书签后，系统自动从选中文本和上下文中提取关键词、生成简短标题和一句话摘要。书签从"原文摘录"变成可识别的信息节点，下次回访时不需要重读全文就能判断书签价值。
Demo：基于文本切分 + 关键词频率规则本地生成
插件：调用大模型 API，返回结构化 JSON（bookmarkTitle / summary / aiNote）
🎯 意图识别（7 类） Demo + 插件
对每条书签自动判断保存意图，分为 7 类：技术实现、产品价值、数据洞察、政策法规、战略建议、个人经历、问题痛点。意图结果驱动书签分组展示和知识点生成策略，用户无需手动打标签。
Demo：本地正则规则匹配（INTENT_ICON 映射表，7 类 + 通用兜底）
插件：大模型对 selectedText + contextText 做语义分类，精度更高
📋 AI 回访简报（Resume Brief） Demo + 插件
当一个页面存在多条书签时，书签库顶部自动生成页面级回访简报：关注焦点、关键要点、后续建议三栏。帮助用户在回访时 10 秒内恢复"上次为什么读这页、接下来该做什么"的记忆，而不是从头扫读。
Demo：遍历书签列表，按意图聚合，模板化生成三栏文本
插件：大模型输入书签元数据列表，生成更自然的自由文本简报
🔗 相关书签联想（Nudge） Demo + 插件
保存新书签时，系统检测与当前页面已有书签的关键词重叠度。若相似度超过阈值，书签库面板弹出提示：「这条书签与『XXX』相关，可一并复盘」。降低内容碎片化，帮助用户在保存时建立关联。
Demo：关键词集合交集计算，重叠 ≥ 2 个词触发 nudge 提示
插件：语义向量相似度检测，召回更准确，可跨页面联想
📝 AI 阅读笔记导出 Demo 已实现
点击「📝 AI 笔记」按钮，系统将当前页面所有书签转化为一份完整的阅读笔记，包含：整页内容摘要 + 每条书签的意图驱动知识点解析。7 种意图各有对应的说明角度——技术实现书签展示实现原理，数据洞察书签展示数字含义，战略建议书签展示行动路径。支持一键复制 Markdown，可直接用于写报告或整理素材，省去手动整理步骤。这是 AnchorNote 从「单条书签理解」到「整页内容整理」的能力延伸。
articleOverview(bms) 读取页面 h1/h2/h3 + 书签关键词，合成整页摘要 · buildKnowledgePoint(bm) 按 intent 字段选择对应解析模板（7 种意图各不同）· buildNotesMarkdown() 组装可复制的 Markdown 格式 · copyNotesMarkdown() 调用 navigator.clipboard.writeText，成功后按钮变绿反馈
⚡ 懒加载书签定位兜底 ★ 核心 Demo 已实现
信息流页面（如腾讯要闻）的部分内容在首屏不渲染。当书签指向未加载的内容时，常规精确文本匹配会返回 null，书签跳转失败。AnchorNote 通过页面级 onJumpFail 钩子实现定位兜底：检测到定位失败 → 自动触发内容加载 → 动画展示卡片渐入过程 → 回调 retryJump() 二次精准定位。整个过程有可见动画，评委能直观看到「系统检测到内容未加载 → 自动处理 → 精准找到」的完整流程。

Part 2 · Chrome 插件端 AI 能力（超越 Demo 的部分）
Extension

插件版调用真实大模型，并新增系统级能力

Chrome 插件通过 AI Proxy（Node/Python 代理服务）调用大模型 API，API Key 保存在代理环境变量中，不写入扩展代码，保护密钥安全。以下是插件版独有或显著增强的 AI 能力。

🧭 AI 语义重定位 ★ 核心 仅插件
页面内容因编辑、改版、广告插入而变化后，精确文本定位失败。插件提取当前页面候选内容块，将原始书签的 selectedText + 上下文做语义嵌入，计算与候选块的余弦相似度，取 Top-1 块作为跳转目标。高置信度时自动跳转，低置信度时提示用户确认，避免误定位。
保存时：选中文本 + context → AI Proxy → 语义嵌入向量写入 Chrome Storage
跳转时：hiText 失败 → 提取页面内容块 → 余弦相似度计算 → Top-1 跳转
👁️ MutationObserver 动态监听 仅插件
插件无需页面手动注册 onJumpFail 钩子，通过 MutationObserver 自动监听 DOM 节点新增事件。书签指向懒加载内容时，插件在后台等待目标文本出现，出现后立即执行定位，对用户完全透明。适配所有信息流、虚拟滚动和动态加载场景。
Content Script 在跳转失败后启动 MutationObserver，监听 childList 变化
目标文本出现后自动调用 hiText 定位，超时（10s）后降级提示用户手动滚动
🔢 出现序号精确定位 仅插件
当同一文本在页面中出现多次（如"重要"、常见短语）时，保存时记录 selectedOccurrenceIndex（第几次出现），跳转时严格对齐到第 N 次，避免多处命中时的错误定位。
保存时：TreeWalker 遍历全文，统计 selectedText 出现次数，记录当前序号
跳转时：多处命中时按序号选取，单处命中时直接定位
🤖 完整结构化 AI 元数据 仅插件（完整版）
插件版每条书签保存时获取完整 AI 结构化输出：bookmarkTitle / summary / intent / keywords / nextAction / aiNote 六字段，阅读断点额外生成 checkpointBrief（前面读了什么 / 停在哪里 / 接下来读什么），均约束为 JSON 格式，UI 稳定渲染不依赖自由文本解析。
Prompt 强制要求 JSON 输出，Background Worker 解析后写入 chrome.storage.local
AI 不可用时自动降级到 shared/meta.js 本地启发式逻辑，保证 Demo 可用
AI Proxy 安全架构： 扩展端（Content Script / Side Panel / Background）只保存代理服务的 URL，不保存 API Key。 所有大模型请求通过 Node.js 或 Python 代理转发，密钥写在代理服务的环境变量中。 代理层支持切换 OpenAI API、腾讯混元大模型 API 或腾讯云智能体开发平台，接口格式 OpenAI-compatible。 API 超时或不可用时，Background Worker 自动切换到本地启发式逻辑，确保核心演示不因网络问题中断。

Part 4 · 核心差异化：AnchorNote 做了什么竞品都没做到
1
书签保存页面内位置，不只是 URL
所有竞品的"收藏"都指向整个页面。AnchorNote 把书签精确到句子级别，保存选中文字、上下文、DOM 位置，回访时直接跳到那句话，不需要用户再次扫读整篇文章。
浏览器书签 / Pocket / Instapaper 均为页面级收藏
2
动态内容定位兜底，业内首创
信息流懒加载、文章改版、广告插入都会导致传统定位失效。AnchorNote 的双层兜底（onJumpFail 自动加载 + AI 语义重定位）使书签在内容变化后仍能找到最相关位置，其他工具遇到这个场景直接失效。
Hypothesis / Weava / Liner 改版后定位失效，无兜底机制
3
意图驱动的 AI 理解，不是泛泛摘要
AnchorNote 对每条书签判断"为什么保存"（7 类意图），并据此生成定制化的知识点解析和下一步建议。Readwise / Liner 的 AI 只做通用摘要，无法区分"这段是数据支撑"还是"这段是政策限制"。
Readwise / Liner AI 摘要无意图分类，所有书签同质化处理
4
AI 阅读笔记一键导出 Markdown
书签不只是标记，而是可以一键转化为完整阅读笔记的原材料。AnchorNote 结合页面结构和书签意图，生成整页摘要 + 逐条知识点，直接可用于写报告、整理素材，省去手动整理环节。
没有任何竞品提供基于阅读意图驱动的笔记导出功能
5
跨页断点续读，平台内无缝衔接
用户在详情页保存断点，切换到频道首页时 CR 横幅自动出现，点击直接跳回。解决了"我知道自己读过但不知道在哪里继续"的问题，尤其适合腾讯新闻等多入口产品生态。
竞品书签只绑定完整 URL，从首页、频道页看不到详情页的断点
6
隐私优先：AI 只处理你选中的内容
Pocket、Readwise、Liner 均需要上传全文到云端才能生成 AI 摘要。AnchorNote 的 AI 只处理用户主动选中并保存的片段，从不扫描整页内容，不在浏览时持续调用 API，数据默认存本地不上云。
Pocket / Readwise / Liner 均上传全文，隐私边界模糊
Part 5 · AI 能力边界说明

🛡️ AnchorNote 的 AI 做了什么，没做什么

  • 只处理用户主动选中并保存的文字片段，从不扫描整页内容
  • 仅在用户主动点击保存/AI 笔记时触发一次 AI 请求，不持续监听页面行为
  • 不生成、改写或替代原文——AI 只帮用户管理自己选的内容
  • 书签数据默认存储在用户本地 localStorage / chrome.storage.local，不上云、不共享
  • API Key 保存在代理服务环境变量，不写入浏览器扩展代码或前端页面
  • AI 不可用时系统自动回退本地规则，核心功能不依赖网络
  • 不做推荐内容、广告定向或任何商业化数据分析
